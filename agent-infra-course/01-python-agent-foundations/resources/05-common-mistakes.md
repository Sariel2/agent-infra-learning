# 01 Python 与 Agent 基础 常见错误

[返回学习地图](../study-map.md) | [返回模块首页](../README.md) | [上一篇：04-sources.md](./04-sources.md)

下面这些错误不是“粗心”，而是这个模块里最常见的思维偏差和工程偏差。每次遇到问题时，你都可以先来这里对照排查。

## 1. 把 route 当成业务层
- 常见表现：HTTP 层里直接写模型调用、参数拼装和错误处理。
- 修正方式：把输入输出 schema 放在 service 边界，route 只负责接收和返回。

## 2. 没有输出 schema
- 常见表现：模型返回文本能看懂，但系统无法稳定消费。
- 修正方式：先定义 `GenerateResponse`，再讨论生成逻辑。

## 3. provider 只是个名字，实际上仍然硬编码实现
- 常见表现：业务逻辑和模型实现依然耦合。
- 修正方式：让 service 依赖 provider 接口而不是具体类。

## 4. 一开始就接真实模型
- 常见表现：测试困难，问题来源混杂。
- 修正方式：先用 fake provider 跑通结构和测试，再换真实实现。

## 5. 测试只测 happy path
- 常见表现：一旦字段缺失或类型变化，系统就无声崩坏。
- 修正方式：至少覆盖 schema 变化、provider 切换和最小错误路径。
