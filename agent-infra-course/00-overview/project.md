# 00 总览与准备项目说明

## 主项目建议
项目名：`Agent Support Runtime`

## 项目一句话描述
做一个面向内部支持场景的 Agent API，能回答知识库问题、调用内部工具查询状态、创建工单，并且具备检索、观测和评测能力。

## 为什么选它
- 业务场景自然，容易理解
- 可以逐阶段叠加复杂度
- 同时覆盖 `tool calling`、`RAG`、`workflow`、`eval`
- 方便后面转成面试项目讲解

## 当前阶段只做这些
- 写项目目标
- 定义阶段边界
- 选定基础技术栈
- 创建最小 repo 结构

## 暂时不要做
- 不要急着做 multi-agent
- 不要急着接数据库和消息队列
- 不要一开始就做前端页面
