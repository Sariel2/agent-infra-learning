# 00 总览与准备 逐步执行说明

## Step 1：读总览与总导图
- 要做什么：读总览与总导图
- 为什么现在做：先建立地图，避免后续陷入局部技术细节。
- 具体动作：通读根 README 与总思维导图，画出你自己的 agent infra 全景图。
- 产出物：全景图初稿
- 完成判断：你能解释 8 个核心模块。
- 常见卡点：把 agent infra 误解成 prompt 工程。
- 如果卡住先检查：检查你是否把 runtime、trace、eval 算进系统。

## Step 2：准备 Python 环境
- 要做什么：准备 Python 环境
- 为什么现在做：统一环境是后续所有代码示例的基础。
- 具体动作：安装 Python 3.11+，选择 uv 或 poetry，确认虚拟环境可用。
- 产出物：可运行的 Python 环境
- 完成判断：你能新建并运行一个最小 Python 项目。
- 常见卡点：系统 Python 与虚拟环境混用。
- 如果卡住先检查：检查 python 与 pip 是否来自同一环境。

## Step 3：建立学习记录
- 要做什么：建立学习记录
- 为什么现在做：课程强调失败案例与迭代证据。
- 具体动作：新建 learning-log、experiments、mistakes、questions 四个记录文件。
- 产出物：记录模板
- 完成判断：你有固定位置记录实验输入、输出和结论。
- 常见卡点：记录太散导致无法复盘。
- 如果卡住先检查：检查记录里是否包含判断依据。

## Step 4：定义主项目
- 要做什么：定义主项目
- 为什么现在做：后续四个工程模块都围绕同一主项目推进。
- 具体动作：用 5 句话写出用户、输入、输出、能力演进路径。
- 产出物：主项目说明
- 完成判断：你能清楚描述整个 6 周要做成什么。
- 常见卡点：范围过大。
- 如果卡住先检查：检查是否第一周就想做 multi-agent、数据库和前端全套。
