# 00 总览与准备 冲刺日历

这一页解决的是“这一模块应该怎么安排时间”，不是“具体每步点哪里”。具体执行顺序看 [steps.md](./steps.md)，这里负责帮你把 8-12 小时的学习时间拆成能落地的节奏。

## 推荐投入
- 总时长：`5-8 小时`
- 推荐拆法：`4 次学习，每次 60-120 分钟`
- 最低保底：如果这周特别忙，至少完成 `Session 1 + Session 2 + Session 4`

## Session 1：建立课程总图
- 建议时长：`60-90 分钟`
- 只做这几件事：
  - 读 [README.md](./README.md)
  - 读 [study-map.md](./study-map.md)
  - 读 [resources/01-concepts-and-principles.md](./resources/01-concepts-and-principles.md)
- 本次目标：
  - 先知道整门课的系统层次和 6 周主线
- 本次产出：
  - 一张你自己的课程总图草稿
  - 一句话版主项目目标

## Session 2：把学习骨架和风险点立住
- 建议时长：`60-90 分钟`
- 只做这几件事：
  - 读 [resources/02-deep-dive.md](./resources/02-deep-dive.md)
  - 读 [resources/05-common-mistakes.md](./resources/05-common-mistakes.md)
  - 扫 [tasks.md](./tasks.md) 和 [project.md](./project.md)
- 本次目标：
  - 明确课程为什么从骨架和记录开始
  - 提前识别项目范围过大、资料太散、迟迟不开始等风险
- 本次产出：
  - 1 份“本周最大阻塞”记录
  - 1 份“主项目范围不过大”的自我确认

## Session 3：跑最小工作区 example
- 建议时长：`90-120 分钟`
- 只做这几件事：
  - 跑 `examples/01-settings-and-logging`
  - 跑 `examples/02-project-smoke-test`
  - 读 [resources/03-code-reading-guide.md](./resources/03-code-reading-guide.md)
- 本次目标：
  - 从一开始就适应“对象化 + 测试化 + 服务化”的节奏
- 本次产出：
  - 至少 1 处你自己做的小改动
  - 至少 1 条你观察到的 readiness 判断

## Session 4：收束到 starter-workspace
- 建议时长：`90-120 分钟`
- 只做这几件事：
  - 读 `module-service/README.md`
  - 跑 `starter-workspace`
  - 做 [review.md](./review.md)
- 本次目标：
  - 把“配置 / readiness / 最小服务边界”收束成统一骨架
- 本次产出：
  - 1 份 readiness 标准
  - 1 份进入第 01 模块前的准备清单

## 时间不够时怎么保底
- 最少完成：
  - `Session 1`
  - `Session 2`
  - `Session 4`
- 可暂时延后：
  - Session 3 里的深度动手改造

## 本模块结束标志
- 你已经知道整门课为什么这样组织
- 你已经有主项目方向
- 你已经准备好在后面每个模块里持续记录和复盘
