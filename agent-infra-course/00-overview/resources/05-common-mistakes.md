# 00 总览与准备 常见错误

[返回学习地图](../study-map.md) | [返回模块首页](../README.md) | [上一篇：04-sources.md](./04-sources.md)

下面这些错误不是“粗心”，而是这个模块里最常见的思维偏差和工程偏差。每次遇到问题时，你都可以先来这里对照排查。

## 1. 把 agent infra 当成 prompt 工程
- 常见表现：看到的都是模型接口，却看不到 runtime、trace、eval 这些更决定成败的层。
- 修正方式：回到总思维导图，重新用“模型/工具/上下文/运行时/观测/评测”六层描述系统。

## 2. 一开始项目范围过大
- 常见表现：第一周就想把多 agent、数据库、前端和部署一起做。
- 修正方式：把第一阶段目标限定为“结构清晰、可运行、可验证”的最小系统。

## 3. 没有学习记录系统
- 常见表现：学到第三周后发现记不住为什么以前某次试验有效。
- 修正方式：建立 learning-log、experiments、mistakes、questions 四类记录文件。

## 4. 把环境问题拖到后面
- 常见表现：后面每个模块都被依赖、虚拟环境和命令问题打断。
- 修正方式：本阶段先把 Python、依赖管理、项目骨架一次性理顺。
