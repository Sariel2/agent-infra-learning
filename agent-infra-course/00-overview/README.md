# 00 总览与准备

这个模块是整套课程的起点。目标不是让你“懂几个名词”，而是先搭起一套稳定的学习和实验骨架，让后面 6 个模块都能在同一套节奏里推进。

## 这个模块解决什么问题
- 让你先建立 agent infra 的全景地图，知道后面每个模块为什么存在。
- 让你把环境、日志、学习记录、实验记录先准备好，避免后面边学边乱。
- 让你先接触课程里统一的工程形状：`讲义 -> example -> module-service -> exercises -> review`。

## 这一模块的最终产出
- 一份你自己的 6 周学习节奏表。
- 一份实验记录模板和失败案例模板。
- 一个最小可运行的 `starter-workspace` 综合服务。
- 一张属于你自己的 agent infra 总体认知图。

## 推荐学习顺序
1. 先看 [mindmap.md](./mindmap.md)，建立整体地图。
2. 再看 [study-map.md](./study-map.md)，确认本模块完整学习路径。
3. 按顺序读讲义：
   [resources/01-concepts-and-principles.md](./resources/01-concepts-and-principles.md)
   [resources/02-deep-dive.md](./resources/02-deep-dive.md)
   [resources/03-code-reading-guide.md](./resources/03-code-reading-guide.md)
   [resources/04-sources.md](./resources/04-sources.md)
   [resources/05-common-mistakes.md](./resources/05-common-mistakes.md)
4. 跟着 [steps.md](./steps.md) 跑 example 和综合服务。
5. 完成 [exercises.md](./exercises.md) 和 [review.md](./review.md)。

## 你在这里最该学会什么
- 知道 agent infra 不只是“调模型”，而是一个带运行时、工具、观测和评测的系统。
- 习惯先搭结构、再做能力、最后再追求效果。
- 习惯把学习过程记录成可复盘、可验证的工程过程。

## 通关标准
- 你能用自己的话解释这套课程 7 个模块分别在解决什么问题。
- 你已经写好自己的学习节奏、实验模板和失败模板。
- 你能运行并解释 [module-service/app/main.py](./module-service/app/main.py) 和 [module-service/app/service.py](./module-service/app/service.py)。
- 你知道后续每个模块都要沿用同一套学习节奏。
