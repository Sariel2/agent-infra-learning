# Final Assessment

这是课程结尾的混合版总测验。它不是一个孤立考试，而是整套课程的最后一轮复习和验收。你做题时应该频繁回到各模块讲义、example、综合服务和复盘记录，而不是把它当成纯记忆测试。

## 这个考核在检验什么
- 你是否真的理解核心概念和边界，而不是只记住术语。
- 你是否能把工程问题拆成排查顺序、证据体系和设计决策。
- 你是否能把课程里的项目讲清楚，并组织成可展示材料。

## 做之前建议先满足这些条件
- 7 个模块都至少跑过一次 example 和 `module-service`。
- 每个模块的 [exercises.md](../00-overview/exercises.md) 和 [review.md](../00-overview/review.md) 都做过。
- 你手上已经有自己的失败案例记录、实验模板和项目讲稿草稿。

## 考核入口
- 题目：[final-exam.md](./final-exam.md)
- 参考答案：[final-exam-answers.md](./final-exam-answers.md)
- 评分标准：[scoring-rubric.md](./scoring-rubric.md)

## 建议做题顺序
1. 先做选择题和判断题，快速暴露概念边界问题。
2. 再做简答题，检查你能不能把概念落到工程语言。
3. 接着做场景分析题和系统设计题，看你是否具备排查和设计能力。
4. 最后做项目讲解题，把前面的项目真正组织成转型材料。

## 建议时长
- 基础题：40-50 分钟
- 简答与场景题：60-90 分钟
- 系统设计与项目讲解：40-60 分钟
- 对答案与复盘：60 分钟

## 做题时最重要的原则
- 不要只写结论，要写为什么这样判断。
- 尽量回链到具体模块、具体讲义或具体代码。
- 场景题一定要有排查顺序。
- 系统设计题一定要写边界、失败恢复和测试思路。
- 项目讲解题一定要讲失败案例和优化动作。

## 如果分数不理想，应该怎么补
- 基础题失分多：
  回看各模块 `resources/01-concepts-and-principles.md`
- 场景题失分多：
  回看各模块 `resources/02-deep-dive.md`、`project.md`
- 系统设计题失分多：
  回到 `module-service/README.md` 和服务代码，重新跑一遍综合项目
- 项目讲解题失分多：
  回到 [../06-career-transition-and-interview/README.md](../06-career-transition-and-interview/README.md)

## 什么时候说明这门课已经真正学完
- 你做完整套考核后，能明确说出自己最强和最弱的部分。
- 你知道弱项应该回哪个模块补，而不是重新泛读全套内容。
- 你能把课程主项目讲成一段可信的工程故事。
- 你已经准备好把这套课程里的项目拿去写简历、讲项目、做面试。
