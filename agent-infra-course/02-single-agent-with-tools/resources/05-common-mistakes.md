# 02 单 Agent 与 Tools 常见错误

[返回学习地图](../study-map.md) | [返回模块首页](../README.md) | [上一篇：04-sources.md](./04-sources.md)

下面这些错误不是“粗心”，而是这个模块里最常见的思维偏差和工程偏差。每次遇到问题时，你都可以先来这里对照排查。

## 1. 工具描述过泛
- 常见表现：模型反复选错工具。
- 修正方式：让工具名字、描述、输入字段都直接对应具体任务。

## 2. 失败返回结构不统一
- 常见表现：系统难以判断失败类型，也难以做 trace 聚合。
- 修正方式：统一 ToolResult 或等价结构。

## 3. 把所有流程都交给模型
- 常见表现：系统波动大、难调试。
- 修正方式：把稳定步骤固定下来，只保留必要的决策点。

## 4. 没有轨迹记录
- 常见表现：出了问题只能猜。
- 修正方式：从一开始记录 tool、args、result、status、latency。

## 5. 把多工具误以为 multi-agent
- 常见表现：架构判断失焦。
- 修正方式：先分清工具数量和 agent 数量是两个维度。
