# 03 RAG、Memory 与 Knowledge 关键资料重点与总结

## 1. OpenAI File Search
- 为什么值得读：帮助你以最小工程成本建立 hosted RAG 成功体验。
- 重点看哪里：文件接入、检索、引用返回。
- 一句话总结：hosted retrieval 适合快速验证知识问答价值。
- 读完你应该会：你能知道最短路径如何做知识问答。
- 阅读说明：建议先读。

## 2. OpenAI Embeddings Guide
- 为什么值得读：理解向量化在检索链路里的位置。
- 重点看哪里：embedding 角色与使用场景。
- 一句话总结：embedding 只是检索链路的一环。
- 读完你应该会：你能理解为什么 RAG 效果不只取决于 embedding 模型。
- 阅读说明：先看概念即可。

## 3. Qdrant Quickstart
- 为什么值得读：帮助你理解自建向量库的最小组件。
- 重点看哪里：collection、point、query。
- 一句话总结：自建向量库带来控制力，也带来维护成本。
- 读完你应该会：你能比较 hosted 与 self-hosted 的权衡。
- 阅读说明：推荐第二遍实操时读。

## 4. LlamaIndex
- 为什么值得读：观察 ingestion、index 和 query pipeline 的抽象方式。
- 重点看哪里：数据接入和检索抽象。
- 一句话总结：框架能帮助组织管道，但前提是你理解基础链路。
- 读完你应该会：你能用框架视角复盘自己的实现。
- 阅读说明：选读。
