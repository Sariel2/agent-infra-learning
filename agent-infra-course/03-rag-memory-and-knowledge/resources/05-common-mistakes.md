# 03 RAG、Memory 与 Knowledge 常见错误

[返回学习地图](../study-map.md) | [返回模块首页](../README.md) | [上一篇：04-sources.md](./04-sources.md)

下面这些错误不是“粗心”，而是这个模块里最常见的思维偏差和工程偏差。每次遇到问题时，你都可以先来这里对照排查。

## 1. 把 RAG 等同于向量库
- 常见表现：过度关注存储实现，忽视 citation 和失败分析。
- 修正方式：先把检索、引用和回答边界理清，再考虑底层选型。

## 2. 没有 citation
- 常见表现：答错时无法定位问题来源。
- 修正方式：把 citation 作为响应模型的一部分。

## 3. 把完整对话都当 memory
- 常见表现：上下文污染严重。
- 修正方式：对短期记忆设置长度上限或摘要规则。

## 4. 不区分检索 miss 和 synthesis drift
- 常见表现：优化动作无效。
- 修正方式：失败记录里单独标注错误层级。

## 5. 一开始就上复杂框架
- 常见表现：链路还没懂，复杂度先上来了。
- 修正方式：先用本地最小实现看清数据流。
