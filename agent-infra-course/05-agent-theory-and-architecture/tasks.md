# 05 Agent 原理与架构 冲刺日历

这一页把第 05 模块拆成一周“架构判断训练”。重点不是再学一堆新术语，而是把你前面做过的系统重新抽象成稳定判断原则。

## 推荐投入
- 总时长：`6-10 小时`
- 推荐拆法：`4 次学习，每次 90-150 分钟`
- 最低保底：如果时间紧，至少完成 `Session 1 + Session 3 + Session 4`

## Session 1：先把 pattern 读回控制流问题
- 建议时长：`90 分钟`
- 只做这几件事：
  - 读 [resources/01-concepts-and-principles.md](./resources/01-concepts-and-principles.md)
  - 读 [resources/05-common-mistakes.md](./resources/05-common-mistakes.md)
- 本次目标：
  - 明确 pattern 的本质是控制流设计
- 本次产出：
  - 1 条“为什么 multi-agent 不是默认升级路线”的说明

## Session 2：把复杂度预算思维建立起来
- 建议时长：`90 分钟`
- 只做这几件事：
  - 读 [resources/02-deep-dive.md](./resources/02-deep-dive.md)
  - 看 [resources/04-sources.md](./resources/04-sources.md)
- 本次目标：
  - 建立“为什么不用另一个方案”的判断框架
- 本次产出：
  - 1 份边界判断问题清单

## Session 3：跑 pattern comparison 和 boundary checker
- 建议时长：`120-150 分钟`
- 只做这几件事：
  - 跑 `examples/01-pattern-comparison`
  - 跑 `examples/02-boundary-checker`
  - 各做 1 个小改动
- 本次目标：
  - 把模式差异和边界推荐从抽象拉回代码
- 本次产出：
  - 1 个新 pattern 或新维度
  - 1 条新增推荐逻辑

## Session 4：读懂 architecture-lab 并准备转型表达
- 建议时长：`120-150 分钟`
- 只做这几件事：
  - 读 [project.md](./project.md)
  - 跑 `architecture-lab`
  - 做 [review.md](./review.md)
- 本次目标：
  - 把架构判断沉淀成后面可讲述、可面试的材料
- 本次产出：
  - 1 份模式比较说明
  - 1 份“为什么不用另一个方案”的回答骨架

## 时间不够时怎么保底
- 最少完成：
  - `Session 1`
  - `Session 3`
  - `Session 4`
- 可暂时压缩：
  - Session 2 的来源深挖

## 本模块结束标志
- 你已经不只是会做系统，还开始会解释系统
- 你已经准备好进入最后的项目包装和面试模块
