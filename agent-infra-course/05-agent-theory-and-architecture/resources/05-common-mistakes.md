# 05 Agent 原理与架构 常见错误

[返回学习地图](../study-map.md) | [返回模块首页](../README.md) | [上一篇：04-sources.md](./04-sources.md)

下面这些错误不是“粗心”，而是这个模块里最常见的思维偏差和工程偏差。每次遇到问题时，你都可以先来这里对照排查。

## 1. 把所有系统都叫 ReAct
- 常见表现：模式判断失去意义。
- 修正方式：回到控制流和状态流去区分模式。

## 2. 模式学习脱离项目
- 常见表现：会背术语，但不会落地判断。
- 修正方式：始终把模式映射到自己的主项目。

## 3. 把多工具误判成 multi-agent
- 常见表现：架构复杂度被高估。
- 修正方式：先确认是否真的存在多个相互独立的执行体。

## 4. 把 workflow 和 agent 对立起来
- 常见表现：做选型时陷入非此即彼。
- 修正方式：优先考虑 workflow 包 agent 的混合式结构。

## 5. 只谈适用场景，不谈失败成本
- 常见表现：判断不够工程化。
- 修正方式：任何模式选择都要说明失败时会付出什么代价。
