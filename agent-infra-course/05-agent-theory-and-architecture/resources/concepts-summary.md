# 05 Agent 原理与架构 概念总结

    ## 关键概念
    - ReAct：交替推理与行动的模式。
- Router：先分流，再交给不同处理路径。
- Plan-and-Execute：先生成计划，再按计划执行。
- Supervisor：上层协调多个执行体的模式。

    ## 容易混淆
    - 看起来能跑的 demo 不等于工程能力已经建立。
    - 会背术语不等于能做边界判断。
    - 有日志不等于有 trace；有试玩不等于有 eval。

    ## 判断原则
    - 优先做最小可运行版本，再补复杂度。
    - 所有知识点最终都要回到 examples 或 module-service 上验证。
    - 每个模块至少保留 1 条失败案例和 1 条改进思路。
