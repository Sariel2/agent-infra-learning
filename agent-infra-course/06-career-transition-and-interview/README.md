# 06 转型、简历与面试

这个模块会把前面所有工程产出整理成可展示、可面试、可转型的表达资产。重点不是再学新技术，而是把“做过”变成“能讲清、讲可信、讲出价值”。

## 这个模块解决什么问题
- 为什么很多人明明做过项目，却在面试里讲不明白。
- 为什么“我接过大模型”不足以支撑 agent infra 转型。
- 为什么项目故事、失败案例、优化证据要一起准备。

## 这一模块的最终产出
- 两个可运行 example：
  [examples/01-project-story-api](./examples/01-project-story-api/README.md)
  [examples/02-interview-qa-pack](./examples/02-interview-qa-pack/README.md)
- 一个最小 `showcase-pack` 综合服务：
  [module-service/README.md](./module-service/README.md)
- 一版业务向项目描述、一版 infra 向项目描述、一版 3 分钟项目讲稿。

## 推荐学习顺序
1. 先看 [mindmap.md](./mindmap.md)，确认表达资产的组成。
2. 再看 [study-map.md](./study-map.md)，明确本模块主线。
3. 按顺序读讲义：
   [resources/01-concepts-and-principles.md](./resources/01-concepts-and-principles.md)
   [resources/02-deep-dive.md](./resources/02-deep-dive.md)
   [resources/03-code-reading-guide.md](./resources/03-code-reading-guide.md)
   [resources/04-sources.md](./resources/04-sources.md)
   [resources/05-common-mistakes.md](./resources/05-common-mistakes.md)
4. 跟着 [steps.md](./steps.md) 跑 example，再做综合服务。
5. 完成 [exercises.md](./exercises.md) 和 [review.md](./review.md)。

## 你在这里最该学会什么
- 把项目讲成“目标、设计、失败、优化、结果”这条完整链。
- 把业务价值和 infra 价值分开表达。
- 把前面模块里的真实失败和真实取舍组织成可信材料。

## 通关标准
- 你能讲清自己的课程主项目，而不是只罗列技术名词。
- 你能写出一条业务版简历描述和一条 infra 版简历描述。
- 你能运行并解释 [module-service/app/service.py](./module-service/app/service.py) 里的 `build_story()` 和 `answer()`。
- 你已经准备好做课程总考核：[assessment/README.md](../assessment/README.md)。

## 完成后去哪
- 总考核入口：[assessment/README.md](../assessment/README.md)
- 在做总考核前，建议先回看前 5 个模块的 [review.md](./review.md) 和你自己的失败案例记录。
