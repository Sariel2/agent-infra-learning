# 06 转型、简历与面试 关键资料重点与总结

## 1. 自己的项目 README 与图
- 为什么值得读：这是你最有说服力的材料。
- 重点看哪里：项目目标、架构、取舍、结果。
- 一句话总结：自己的项目材料比背别人的案例更重要。
- 读完你应该会：你能整理出统一项目故事。
- 阅读说明：必读。

## 2. OpenAI 官方 agent 资料回顾
- 为什么值得读：帮助把自己的实践放回行业语境。
- 重点看哪里：tools、tracing、runtime。
- 一句话总结：你的项目不是孤立 demo，而是沿着官方主脉络演进。
- 读完你应该会：你能解释自己的技术路线为什么合理。
- 阅读说明：选重点回顾。

## 3. LangGraph 回顾
- 为什么值得读：帮助回答 workflow orchestration 选型问题。
- 重点看哪里：为什么需要显式 workflow。
- 一句话总结：选型表达来自真实工程痛点，而不是流行词。
- 读完你应该会：你能把自己的工程取舍讲清楚。
- 阅读说明：配合自己的项目一起看。
