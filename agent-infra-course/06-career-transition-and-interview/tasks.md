# 06 转型、简历与面试任务安排

## Day 1：项目包装
- 重写主项目 README
- 补系统设计图
- 补运行截图或 trace 截图

## Day 2：项目讲稿
- 写 3 分钟版本讲稿
- 写 10 分钟版本讲稿
- 准备 3 个关键 trade-off 故事

## Day 3：简历改写
- 写一版项目描述，强调业务价值
- 写一版项目描述，强调 infra 能力
- 把 Java 后端经验翻译成 AI Infra 相关表达

## Day 4：面试题准备
- 准备 `Agent vs Workflow`
- 准备 `RAG 失败分析`
- 准备 `tool calling 稳定性`
- 准备 `observability 与 eval`

## Day 5：模拟问答
- 自问自答或录音演练
- 调整表述中的空话和套话
- 输出最终转型材料

## 退出条件
- 有可展示项目链接或本地演示材料
- 有可直接放进简历的项目描述
- 有一套稳定回答框架
